BPMN-модель процесса обработки заявок (Кейс GreenTech)

Цель проекта:
Моделирование процесса обработки заявок на услуги «As-Is» (как есть) для последующей автоматизации и внедрения CRM-системы. Проект демонстрирует навык работы с многоуровневыми согласованиями и разделением зон ответственности между разными ролями.

Описание процесса:
В модели описан цикл обработки клиентской заявки: от заполнения веб-формы на сайте до получения клиентом финального коммерческого предложения. 
В процессе задействованы три ключевых участника, разделенных по дорожкам (Lanes):
•  Клиент: Инициирует запрос и согласует промежуточные этапы.
•  Сотрудник (Back-office): Проводит первичную проверку и готовит документацию.
•  Руководство: Отвечает за финальное утверждение условий сделки.

Основные сценарии:

•  Happy Path (Основной сценарий):
  1. Клиент заполняет веб-форму на сайте компании.
  2. Сотрудник оперативно проверяет данные и формирует предварительное предложение.
  3. Клиент подтверждает условия предварительного предложения.
  4. Руководство компании одобряет финальную версию предложения.
  5. Сотрудник отправляет окончательный оффер клиенту.
  6. Процесс успешно завершается.

•  Альтернативные сценарии и циклы:
  •  Отказ руководства: Если руководство не согласовало условия, процесс возвращается на этап корректировки предложения сотрудником (цикл доработки).
  •  Отказ клиента: Если клиент не согласен с предварительными условиями, процесс завершается без перехода к руководству.
  •  Некорректные данные: Если заявка не прошла первичную проверку сотрудником, процесс прекращается на начальном этапе.

Использованные элементы BPMN:
•  Pools & Lanes: Четкое разграничение ответственности между Клиентом, Сотрудником и Руководителем.
•  Exclusive Gateways (X): Для реализации логики ветвления (согласовано/не согласовано).
•  Message Start Event: Запуск процесса при поступлении данных из внешней системы (веб-форма).
•  Sequence Flows: Потоки последовательности действий внутри организации.
