В рамках проекта был разработан целевой процесс обработки HR-запросов, связанных с предоставлением кадровых данных сотрудникам и подразделениям. Целью было выявить основные проблемы текущего процесса и предложить решение, которое снижает риск потери заявок и срыва сроков выполнения.

Задача

Исходный процесс предполагал обработку запросов на кадровую информацию, поступающих из разных каналов. Основные проблемы процесса:

заявки могут теряться;
сроки выполнения запросов могут срываться;
отсутствует прозрачность статуса обращения для заявителя;
сложные запросы требуют ручного сбора данных из нескольких источников.
Решение

В проекте предложено:

централизовать регистрацию всех входящих запросов в единой системе;
разделить запросы на простые и сложные;
назначать ответственного за сложные запросы;
устанавливать срок выполнения запроса;
добавлять уведомления о сроке выполнения и о его переносе;
контролировать просрочку через механизм напоминаний;
закрывать запрос только после подтверждения результата заявителем.
